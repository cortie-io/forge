# Network Exam QA Benchmark Report

## 1. 실험 목적

본 실험의 목적은 네트워크관리사 유형 객관식 문제(120문항)에 대해, 세 가지 추론 방식이
정답률, 해설 품질, 근거 기반성, 지연 시간 측면에서 어떤 차이를 보이는지 정량적으로 비교하는 것이다.

비교 대상은 다음과 같다.

- A (LLM-only): 문제 텍스트만으로 답변 생성
- B (Naive RAG): 문제 기반 검색 후 근거를 포함해 답변 생성
- C (Full Pipeline): 온톨로지 분석(의도/좌표) + RAG + 해설 파이프라인

참고: 본 실험의 C 그룹은 문제 해설 흐름을 강제하기 위해 EXPLAIN_PROBLEM 기준으로 실행되었다.

## 2. 데이터 및 실험 설정

- 평가 문항 수: 120
- 각 실험군 샘플 수: 120
- 공통 입력: 동일 문제 CSV
- 평가 모델: gpt-4o-mini (LLM-as-a-Judge)
- 평가 조건: require-judge=1 (API 키 필수)
- 생성/판정 설정: temperature=0

## 3. 평가 방법 개요

평가는 문항 단위로 이루어지며, 각 문항에서 아래 정보를 결합해 지표를 계산한다.

- 정답 라벨(답)
- 모델 선택지(AI_정답)
- 해설 텍스트(해설)
- 근거 문서(RAG_근거, RAG_근거_JSON)
- 예측 좌표(pred_subject 등)

최종적으로 문항별 점수를 실험군 단위 평균으로 집계한다.

## 4. 지표 정의 및 계산식

### 4.1 정답 관련

1) 정답정확도 (정답률의 원지표)

$$
\text{AnswerCorrectness}_i = \mathbb{1}[\hat{y}_i = y_i]
$$

- 일치 시 1, 불일치 시 0
- 주의: 현재 구현은 보기 값을 parse_choice로 정규화해 `1`, `1.0`, `정답은 1번` 등을 동일하게 처리한다.

2) 정답률(Avg)

$$
\text{Accuracy} = \frac{1}{N}\sum_{i=1}^{N}\text{AnswerCorrectness}_i
$$

### 4.2 근거 기반성 관련

1) 근거충실도 (Faithfulness)

해설을 문장 단위로 분할한 뒤, 각 문장이 근거 문서와 충분히 겹치는지 확인한다.

$$
\text{Faithfulness} = \frac{N_{\text{supported sentences}}}{N_{\text{all sentences}}}
$$

- 문장-근거 최대 토큰중첩비가 0.22 이상이면 supported로 판정

2) 근거정밀도 (Context Precision)

근거 문서 순위에 로그 감쇠 가중치를 주어 계산한다.

$$
w_k = \frac{1}{\log_2(k+2)},\quad
r_k = \min\left(1,\;2.2\cdot\text{overlap}(q, d_k)\right)
$$

$$
\text{ContextPrecision} = \frac{\sum_k w_k r_k}{\sum_k w_k}
$$

- $q$: 문제 + 정답 텍스트 + 해설 앞부분(최대 220자) 앵커
- $d_k$: k번째 근거 문서

3) 환각발생률

$$
\text{HallucinationRate} = 1 - \text{Faithfulness}
$$

- 낮을수록 좋다.

### 4.3 해설 논리/표현 관련

1) 오답소거논리점수

- 해설 내 [오답 포인트] 섹션에서 1~4번 오답 분석 라인을 찾음
- 아래 둘 중 하나를 만족하면 해당 보기를 유효한 소거로 판정
- 기술 단서어 포함(예: 포트, 계층, 프로토콜, 라우팅, 주소, TCP/UDP/IP 등)
- 해당 보기 텍스트와 토큰 중첩비 >= 0.12

$$
\text{DistractorScore} = \frac{N_{\text{valid eliminations}}}{4}
$$

2) 전문용어정확도

- 해설에서 전문용어 후보를 추출한 뒤 용어 사전(raw structure 기반) 매칭률 계산

$$
\text{TermPrecision} = \frac{N_{\text{matched terms}}}{N_{\text{candidate terms}}}
$$

- 후보가 없으면 1.0 처리

3) 가독성/정확성/완결성 (LLM Judge)

- LLM 판정기로 각 항목을 1~5점으로 채점
- 항목: accuracy, completeness, clarity
- 최종 값은 [1, 5] 구간으로 clipping

### 4.4 온톨로지/구조 관련

1) 좌표 정확도(좌표일치도)

$$
\text{CoordMatch}_i = \mathbb{1}[\text{pred\_subject}_i = \text{expected\_subject}_i]
$$

2) 계층이탈률

- 해설 토큰이 기대 과목 외 다른 과목 vocabulary와 3개 이상 겹치면 이탈 hit
- 과목 수 정규화 평균으로 비율화

$$
\text{DeviationRate} = \frac{N_{\text{deviation hits}}}{N_{\text{other subjects}}}
$$

## 6. 결과 해석

### 6.1 전체 성능 순위(핵심 관점)

- 정답률: A > C >> B
- 해설품질(LLM 평가): C > A > B
- 지연시간: A < C < B

### 6.2 실험군별 해석

1) A (LLM-only)
- 장점: 가장 빠르고 정답률이 가장 높다.
- 한계: 근거정밀도 0, 환각발생률 1.0으로 기록됨.
- 해석 주의: A는 근거 문서를 쓰지 않는 구조이므로 근거 기반 지표에서 구조적으로 불리하다.

2) B (Naive RAG)
- 장점: 근거 기반 지표(근거정밀도, 환각률)는 A보다 개선.
- 한계: 정답률이 크게 하락(57.5%), 지연도 가장 길다.
- 해석: 검색은 붙었지만 검색 품질/정답 선택 연계가 약해 실효성이 떨어진 상태.

3) C (Full Pipeline)
- 장점: 정답률 71.67%로 A에 근접, LLM 품질 점수(정확성/완결성/가독성)는 3그룹 중 최고.
- 한계: 좌표 정확도와 계층이탈률은 아직 개선 여지 존재.
- 해석: 정확도-품질-속도 균형 관점에서 현재 가장 운영 친화적인 구성.

**근거충실도(Faithfulness) 심층 해석:**

C의 근거충실도(0.389)와 B(0.399)는 수치상 거의 동등하다. 이는 "얼마나 많은 근거 문장을 인용했는가"의 관점에서 두 시스템이 비슷한 수준임을 의미한다.

그러나 Faithfulness 지표의 본질적 의미는 인용량이 아니라 **"검색된 근거에 기반하여 얼마나 충실하게 설명하는가"**이다. 하나의 근거 문장만 사용하더라도 그 내용에 완전히 충실한 설명이 생성된다면 높은 충실도를 가진다.

C가 B와 같은 충실도를 보이면서도 LLM 해설 품질(정확성 4.36, 완결성 4.48, 가독성 4.52)이 유의미하게 높은 이유는, 온톨로지 좌표 기반 문서 선택으로 **더 적합한 근거**를 검색했기 때문이다. 즉, C는 B보다 인용량이 적더라도 주제 정합성 높은 근거를 선택함으로써 설명의 완성도를 높였다.

**현재 Faithfulness 수준(0.389)의 평가:**
본 실험에서 C와 B 모두 Faithfulness 약 0.40 수준으로, 해설 문장의 약 40%만이 검색 근거에 직접적으로 기반한다고 볼 수 있다. 이는 논문 투고 기준에서 보면 **개선이 필요한 수준**이다. 해설의 절반 이상이 모델의 내재 지식(파라메트릭 지식)에 의존하고 있으며, 이는 할루시네이션 위험과 설명 재현성 저하의 원인이 된다.

## 7. 현재 성능 수준 평가

실험 결과를 종합하면, 본 시스템은 "중상급 프로토타입" 수준으로 판단된다.

- 강점: C 그룹 기준 해설 품질이 높고, 정답률도 실사용 가능한 수준에 근접
- 약점: 정답률 상한(80%대) 미도달, 온톨로지 좌표 정합성과 계층 이탈 제어가 아직 미흡

실무 기준으로는, 내부 베타/파일럿에는 투입 가능하나, 고신뢰 자동 채점/교육 보조 서비스의 본 운영 전에는 추가 개선이 필요하다.

- A는 비근거형 구조라 근거정밀도/환각률 지표가 본질적으로 불리하게 나온다.
- 따라서 A vs B/C 비교 시, 정답률/지연과 근거지표를 분리 해석해야 한다.

2) LLM-as-a-Judge 편향
- 판정 모델 자체 편향 가능성 존재
- 다만 동일 모델/동일 프롬프트를 전 그룹에 적용해 상대 비교 일관성은 확보

3) 데이터셋 범위
- 단일 120문항 세트 기반 결과로, 도메인 확장 일반화 성능은 별도 검증 필요

## 9. 개선 권고안

1) C 그룹 정답률 상향(1차 목표 75%, 2차 목표 80%)
- 좌표 정확도 개선: pred_subject 정합 강화
- 계층 이탈 억제: 해설 생성 시 타 과목 토큰 페널티 또는 후처리 규칙

2) B 그룹 구조 보강
- search_query 품질 개선 및 근거 재랭킹 도입
- 근거-정답 연결 강화(정답 후보 검증 단계 추가)

3) 평가 체계 고도화
- 과목별/난이도별 성능 분해 리포트 자동화
- 인간 평가 샘플링(예: 120문항 중 20문항) 병행하여 Judge 신뢰도 점검

4) 근거충실도(Faithfulness) 향상 방안 — 논문 투고 대비

현재 B·C 모두 Faithfulness ≈ 0.39 수준으로, 논문에서 "근거 기반 해설 시스템"을 주장하기 위해서는 0.60 이상을 목표로 삼는 것이 권고된다. 아래 기법을 단계적으로 적용할 수 있다.

**a) 프롬프트 설계 개선 (즉각 적용 가능)**
- 현재 프롬프트는 근거 문서를 참조는 하지만 "반드시 근거 문장에 기반하여 설명하라"는 명시적 지시가 부족하다.
- Chain-of-Thought(CoT) + Evidence Anchoring 기법 적용: "먼저 제공된 근거에서 핵심 문장을 직접 인용하고, 그 내용을 확장하여 설명하라"는 구조를 프롬프트에 삽입한다.
- 기대 효과: Faithfulness 0.15~0.25p 향상 가능.

**b) 근거 문서 청크 품질 개선 (RAG 파이프라인 개선)**
- 현재 ChromaDB에 저장된 청크 단위가 너무 크거나 작아 해설 생성 시 비효율적 참조가 발생할 수 있다.
- Sentence-level chunking 또는 Proposition-based chunking 적용: 각 청크가 하나의 완결된 명제를 포함하도록 분해한다.
- 기대 효과: 검색 정밀도 향상 → 충실도 개선.

**c) 근거 인용 강제 후처리 (Post-processing)**
- 해설 생성 후, 검색 근거와의 토큰 오버랩이 임계값 이하인 문장을 "미지원 문장"으로 플래그 처리하고 재생성 요청하는 검증 루프 추가.
- Attribution-based Reranking: 생성된 해설 후보 중 Faithfulness 점수가 가장 높은 것을 선택하는 샘플링 전략(Best-of-N).

**d) Fine-tuning / RLHF (장기 과제)**
- 근거 문서에 충실한 해설 예시를 선호 데이터셋으로 구성하여 DPO(Direct Preference Optimization) 또는 RLHF 방식으로 모델을 미세조정한다.
- 논문에서 이를 "향후 과제(Future Work)"로 명시하면 기여도와 완성도를 동시에 높일 수 있다.

**e) 평가 지표 보강 (논문 신뢰도 향상)**
- 현재 Faithfulness 측정은 토큰 오버랩(threshold 0.22) 기반으로, 의미 유사도를 완전히 포착하지 못할 수 있다.
- NLI(Natural Language Inference) 기반 Faithfulness 측정 병행 권고: 예) TRUE, MiniCheck, AlignScore 등의 오픈소스 NLI 모델을 활용해 "근거 문서가 해설 문장을 entail하는가"를 판별.
- 이 두 가지 측정값을 함께 보고하면 논문의 평가 방법론 신뢰성이 크게 향상된다.

## 10. 결론

- 최고 정답률/최저 지연은 A가 기록했지만, 근거 기반 설명력까지 포함하면 C가 가장 균형적이다.
- 현재 운영 후보는 C를 우선 권장한다.
- 단, 목표 성능(안정적 80%대 정답률) 달성을 위해 좌표 정합 및 근거-정답 연결 개선이 필수적이다.
